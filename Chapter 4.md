# บทที่ 4 - หน่วยความจำแคช (Cache Memory)

หน่วยความจำแคชเป็นองค์ประกอบสำคัญของสถาปัตยกรรมคอมพิวเตอร์สมัยใหม่ โดยถูกออกแบบมาเพื่อเพิ่มประสิทธิภาพการทำงานของระบบคอมพิวเตอร์ให้สามารถเข้าถึงข้อมูลได้รวดเร็วยิ่งขึ้น เนื่องจากหน่วยประมวลผลกลาง (CPU) มีความเร็วสูงมากเมื่อเทียบกับหน่วยความจำหลัก จึงจำเป็นต้องมีหน่วยความจำความเร็วสูงที่ทำหน้าที่เป็นตัวกลางระหว่างโปรเซสเซอร์กับหน่วยความจำหลัก ซึ่งก็คือแคชเมมโมรี

# 4.1 - ภาพรวมของระบบหน่วยความจำคอมพิวเตอร์ (Computer Memory System Overview)

ระบบหน่วยความจำของคอมพิวเตอร์มีความหลากหลายทั้งในด้านเทคโนโลยี โครงสร้าง ประสิทธิภาพ และต้นทุน จึงไม่สามารถใช้หน่วยความจำเพียงชนิดเดียวเพื่อตอบสนองความต้องการทั้งหมดของระบบได้ ด้วยเหตุนี้จึงเกิดแนวคิดของลำดับชั้นหน่วยความจำ (Memory Hierarchy) ซึ่งเป็นการจัดเรียงหน่วยความจำหลายประเภทเข้าด้วยกันเพื่อให้ได้สมดุลระหว่างความเร็ว ความจุ และต้นทุน

โดยทั่วไปลำดับชั้นของหน่วยความจำประกอบด้วย

- รีจิสเตอร์ (Registers)
- แคชเมมโมรี (Cache Memory)
- หน่วยความจำหลัก (Main Memory)
- หน่วยความจำสำรอง เช่น ฮาร์ดดิสก์ หรือเทปแม่เหล็ก

เมื่อพิจารณาลงไปตามลำดับชั้นของหน่วยความจำ จะพบแนวโน้มสำคัญ ได้แก่

- ความจุของหน่วยความจำเพิ่มขึ้น
- ราคาต่อบิตลดลง
- ความเร็วในการเข้าถึงลดลง
- ความถี่ที่โปรเซสเซอร์เข้าถึงลดลง

โครงสร้างดังกล่าวช่วยให้ระบบคอมพิวเตอร์สามารถใช้หน่วยความจำความเร็วสูงสำหรับข้อมูลที่ใช้งานบ่อย และใช้หน่วยความจำที่มีความจุสูงแต่ราคาถูกสำหรับการเก็บข้อมูลจำนวนมาก

คุณลักษณะสำคัญของระบบหน่วยความจำสามารถจำแนกได้ดังนี้

- ตำแหน่งของหน่วยความจำ  
  - หน่วยความจำภายใน (Internal Memory) เช่น รีจิสเตอร์ แคช และหน่วยความจำหลัก  
  - หน่วยความจำภายนอก (External Memory) เช่น ดิสก์ เทป และสื่อบันทึกข้อมูลอื่น ๆ

- ความจุ (Capacity)  
  วัดเป็นจำนวนไบต์หรือจำนวนคำ (Word)

- หน่วยการถ่ายโอนข้อมูล (Unit of Transfer)  
  - Word  
  - Block

- วิธีการเข้าถึงข้อมูล (Access Method)  
  - Sequential Access  
  - Direct Access  
  - Random Access  
  - Associative Access

- สมรรถนะ (Performance)  
  - Access Time  
  - Cycle Time  
  - Transfer Rate

# 4.2 - หลักการของหน่วยความจำแคช (Cache Memory Principles)

หน่วยความจำแคชถูกออกแบบมาเพื่อลดเวลาเฉลี่ยในการเข้าถึงหน่วยความจำ โดยรวมข้อดีของหน่วยความจำความเร็วสูงขนาดเล็กเข้ากับหน่วยความจำขนาดใหญ่ที่มีราคาถูกกว่า

แคชจะเก็บสำเนาของข้อมูลบางส่วนจากหน่วยความจำหลักไว้ เมื่อโปรเซสเซอร์ต้องการอ่านข้อมูล ระบบจะตรวจสอบก่อนว่าข้อมูลนั้นอยู่ในแคชหรือไม่ ซึ่งสามารถเกิดได้สองกรณี คือ

- Cache Hit  
  ข้อมูลที่ต้องการอยู่ในแคช ทำให้สามารถส่งข้อมูลให้โปรเซสเซอร์ได้ทันที

- Cache Miss  
  ข้อมูลไม่อยู่ในแคช จำเป็นต้องอ่านข้อมูลจากหน่วยความจำหลักก่อน แล้วจึงนำข้อมูลนั้นมาเก็บไว้ในแคช

ประสิทธิภาพของแคชเกิดจากหลักการที่เรียกว่า Locality of Reference ซึ่งอธิบายพฤติกรรมการเข้าถึงข้อมูลของโปรแกรม โดยมีลักษณะสำคัญสองประการ ได้แก่

- Temporal Locality  
  ข้อมูลที่เพิ่งถูกใช้งานมีแนวโน้มที่จะถูกใช้งานอีกครั้งในเวลาอันใกล้

- Spatial Locality  
  ข้อมูลที่อยู่ใกล้ตำแหน่งกันในหน่วยความจำมักถูกเข้าถึงต่อเนื่องกัน

ในระบบคอมพิวเตอร์สมัยใหม่ มักใช้แคชหลายระดับเพื่อเพิ่มประสิทธิภาพ ได้แก่

- L1 Cache ซึ่งมีความเร็วสูงที่สุดและมีขนาดเล็ก
- L2 Cache ซึ่งมีขนาดใหญ่ขึ้นและความเร็วลดลงเล็กน้อย
- L3 Cache ซึ่งมีขนาดใหญ่กว่าแต่มีความเร็วต่ำกว่า L1 และ L2

# 4.3 - องค์ประกอบของการออกแบบแคช (Elements of Cache Design)

การออกแบบแคชเมมโมรีมีองค์ประกอบสำคัญหลายประการ ดังนี้

- Cache Addresses  
  การจัดการที่อยู่ของแคชสามารถแบ่งเป็นสองประเภท ได้แก่  
  - Logical หรือ Virtual Cache ซึ่งใช้ที่อยู่เสมือนจากโปรแกรมโดยตรง  
  - Physical Cache ซึ่งใช้ที่อยู่จริงของหน่วยความจำหลัก

- ขนาดของแคช (Cache Size)  
  การกำหนดขนาดของแคชต้องคำนึงถึงความสมดุลระหว่างความเร็ว ต้นทุน และพื้นที่บนชิป แคชที่มีขนาดใหญ่สามารถเก็บข้อมูลได้มากขึ้น แต่ก็อาจเพิ่มความซับซ้อนและเวลาในการเข้าถึง

- วิธีการแมปข้อมูล (Mapping Function)  
  กำหนดว่าบล็อกของหน่วยความจำหลักจะถูกจัดเก็บในตำแหน่งใดของแคช โดยมีวิธีหลักสามรูปแบบ ได้แก่  
  - Direct Mapping ซึ่งบล็อกหนึ่งสามารถเก็บได้เพียงตำแหน่งเดียวในแคช  
  - Associative Mapping ซึ่งบล็อกสามารถเก็บในตำแหน่งใดก็ได้ในแคช  
  - Set-Associative Mapping ซึ่งแบ่งแคชออกเป็นหลายชุด และแต่ละชุดมีหลายบรรทัด

- อัลกอริทึมการแทนที่ (Replacement Algorithm)  
  เมื่อแคชเต็ม ระบบจำเป็นต้องเลือกบล็อกข้อมูลที่จะถูกแทนที่ ตัวอย่างอัลกอริทึมที่ใช้ ได้แก่  
  - LRU (Least Recently Used)  
  - FIFO (First In First Out)  
  - LFU (Least Frequently Used)  
  - Random Replacement

- นโยบายการเขียนข้อมูล (Write Policy)  
  กำหนดวิธีจัดการเมื่อมีการเขียนข้อมูลลงในแคช เช่น  
  - Write Through คือเขียนข้อมูลลงทั้งแคชและหน่วยความจำหลักพร้อมกัน  
  - Write Back คือเขียนข้อมูลเฉพาะในแคชก่อน และจะเขียนกลับไปยังหน่วยความจำหลักเมื่อมีการแทนที่ข้อมูล

- ขนาดบรรทัดของแคช (Line Size)  
  กำหนดจำนวนข้อมูลในแต่ละบล็อกที่ถูกนำเข้าสู่แคชในครั้งเดียว

- จำนวนระดับของแคช  
  ระบบอาจใช้แคชระดับเดียว หรือหลายระดับ เช่น L1 L2 และ L3 รวมทั้งอาจแบ่งเป็น  
  - Unified Cache ซึ่งเก็บทั้งคำสั่งและข้อมูลร่วมกัน  
  - Split Cache ซึ่งแยก Instruction Cache และ Data Cache

# 4.4 - โครงสร้างแคชของ Pentium 4

สถาปัตยกรรมของหน่วยประมวลผล Pentium 4 มีการใช้แคชหลายระดับเพื่อเพิ่มประสิทธิภาพของระบบ โดยประกอบด้วย

- L1 Instruction Cache
- L1 Data Cache
- L2 Cache
- L3 Cache

นอกจากนี้ยังมีองค์ประกอบสำคัญอื่น ๆ ภายในหน่วยประมวลผล เช่น

- หน่วยดึงและถอดรหัสคำสั่ง (Fetch/Decode Unit)
- ระบบประมวลผลแบบ Out-of-Order
- หน่วยประมวลผลคำสั่ง (Execution Units)
- ระบบย่อยหน่วยความจำ (Memory Subsystem)

โครงสร้างดังกล่าวช่วยเพิ่มประสิทธิภาพในการประมวลผลของโปรเซสเซอร์ โดยลดระยะเวลาที่หน่วยประมวลผลต้องรอการเข้าถึงข้อมูลจากหน่วยความจำ ทำให้ระบบคอมพิวเตอร์สามารถทำงานได้รวดเร็วและมีประสิทธิภาพมากยิ่งขึ้น
