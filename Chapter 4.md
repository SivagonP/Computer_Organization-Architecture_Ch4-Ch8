# บทที่ 4 - หน่วยความจำแคช (Cache Memory)
## หน่วยความจำของคอมพิวเตอร์แม้จะดูเหมือนเป็นองค์ประกอบที่มีแนวคิดพื้นฐานไม่ซับซ้อน แต่ในความเป็นจริงแล้วถือเป็นองค์ประกอบที่มีความหลากหลายมากที่สุดในระบบคอมพิวเตอร์ ทั้งในด้านชนิดของหน่วยความจำ เทคโนโลยีที่ใช้ โครงสร้างการจัดระเบียบ ประสิทธิภาพการทำงาน และต้นทุนในการผลิต

ไม่มีเทคโนโลยีหน่วยความจำชนิดใดชนิดหนึ่งที่สามารถตอบสนองความต้องการทั้งหมดของระบบคอมพิวเตอร์ได้อย่างเหมาะสม ดังนั้นระบบคอมพิวเตอร์โดยทั่วไปจึงใช้แนวคิดของ ลำดับชั้นของหน่วยความจำ (Memory Hierarchy) ซึ่งประกอบด้วยหน่วยความจำหลายระดับที่ทำงานร่วมกัน
หน่วยความจำบางส่วนอยู่ภายในระบบและสามารถเข้าถึงได้โดยตรงจากหน่วยประมวลผลกลาง (CPU) ในขณะที่หน่วยความจำบางประเภทอยู่ภายนอกและต้องเข้าถึงผ่านโมดูลอินพุต/เอาต์พุต
บทนี้จะเน้นศึกษาหน่วยความจำภายในระบบ โดยเฉพาะองค์ประกอบสำคัญของระบบคอมพิวเตอร์สมัยใหม่ ได้แก่ หน่วยความจำแคช (Cache Memory)
วัตถุประสงค์การเรียนรู้ของบทนี้
หลังจากศึกษาบทนี้แล้ว ผู้อ่านควรสามารถ
-อธิบายลักษณะสำคัญของระบบหน่วยความจำคอมพิวเตอร์ได้
-อธิบายแนวคิดของลำดับชั้นหน่วยความจำ
-อธิบายหลักการพื้นฐานของหน่วยความจำแคช
-อธิบายองค์ประกอบสำคัญในการออกแบบแคช
-แยกความแตกต่างระหว่าง Direct Mapping, Associative Mapping และ Set-Associative Mapping
-อธิบายเหตุผลในการใช้แคชหลายระดับ
-วิเคราะห์ผลกระทบด้านประสิทธิภาพของระบบหน่วยความจำหลายระดับ
##4.1 ภาพรวมของระบบหน่วยความจำคอมพิวเตอร์ (Computer Memory System Overview)
###ลักษณะของระบบหน่วยความจำ
เพื่อให้เข้าใจระบบหน่วยความจำได้ง่ายขึ้น เราสามารถจำแนกหน่วยความจำตามลักษณะสำคัญหลายประการ ได้แก่
-ตำแหน่งของหน่วยความจำ
-ความจุของหน่วยความจำ
-วิธีการเข้าถึงข้อมูล
-ประสิทธิภาพในการทำงาน
-ลักษณะทางกายภาพของหน่วยความจำ
-การจัดระเบียบของข้อมูล
การจำแนกหน่วยความจำตามลักษณะเหล่านี้ช่วยให้สามารถวิเคราะห์และออกแบบระบบหน่วยความจำที่เหมาะสมกับความต้องการของระบบคอมพิวเตอร์ได้
-การจำแนกหน่วยความจำตามตำแหน่ง (Location)
หน่วยความจำสามารถแบ่งตามตำแหน่งได้เป็นสองประเภทหลัก
-หน่วยความจำภายใน (Internal Memory)
-หน่วยความจำภายนอก (External Memory)
###หน่วยความจำภายใน
หน่วยความจำภายในหมายถึงหน่วยความจำที่อยู่ภายในระบบคอมพิวเตอร์และสามารถเข้าถึงได้โดยตรงจาก CPU ตัวอย่างเช่น
-รีจิสเตอร์ (Registers)
-หน่วยความจำแคช (Cache)
-หน่วยความจำหลัก (Main Memory)
-หน่วยความจำประเภทนี้มีความเร็วสูง แต่มีความจุจำกัด
###หน่วยความจำภายนอก
หน่วยความจำภายนอกหมายถึงอุปกรณ์จัดเก็บข้อมูลที่ใช้สำหรับเก็บข้อมูลระยะยาว เช่น
-ฮาร์ดดิสก์
-SSD
-อุปกรณ์จัดเก็บข้อมูลแบบแม่เหล็ก
-อุปกรณ์จัดเก็บข้อมูลแบบออปติคัล
-หน่วยความจำประเภทนี้มีความจุสูงแต่ความเร็วต่ำกว่า
-การจำแนกหน่วยความจำตามความจุ (Capacity)
###ความจุของหน่วยความจำหมายถึงจำนวนข้อมูลสูงสุดที่สามารถเก็บได้ โดยทั่วไปจะวัดเป็น
-ไบต์ (Byte)
-กิโลไบต์ (KB)
-เมกะไบต์ (MB)
-กิกะไบต์ (GB)
-เทราไบต์ (TB)
ความจุของหน่วยความจำมีผลโดยตรงต่อความสามารถของระบบในการจัดเก็บโปรแกรมและข้อมูล
##วิธีการเข้าถึงข้อมูล (Access Method)
###หน่วยความจำสามารถจำแนกตามวิธีการเข้าถึงข้อมูลได้ดังนี้
-Sequential Access
-Direct Access
-Random Access
-Associative Access
-Sequential Access
ข้อมูลจะถูกเข้าถึงตามลำดับ เช่น เทปแม่เหล็ก ซึ่งต้องอ่านข้อมูลตั้งแต่ต้นจนถึงตำแหน่งที่ต้องการ
##Direct Access

สามารถเข้าถึงตำแหน่งข้อมูลที่ต้องการได้โดยตรง แต่ยังต้องใช้เวลาบางส่วนในการค้นหาตำแหน่ง เช่น ฮาร์ดดิสก์

##Random Access

สามารถเข้าถึงข้อมูลตำแหน่งใดก็ได้ในเวลาใกล้เคียงกัน ตัวอย่างเช่น RAM

##Associative Access

การเข้าถึงข้อมูลโดยใช้เนื้อหาของข้อมูลเป็นตัวกำหนดตำแหน่ง เช่น Content Addressable Memory

ประสิทธิภาพของหน่วยความจำ (Performance)

###ประสิทธิภาพของหน่วยความจำมักพิจารณาจากปัจจัยสำคัญ ได้แก่

-Access Time

-Memory Cycle Time

-Transfer Rate

##Access Time

เวลาที่ใช้ตั้งแต่เริ่มร้องขอข้อมูลจนกระทั่งข้อมูลพร้อมใช้งาน

##Memory Cycle Time

เวลาที่ต้องใช้ก่อนที่จะสามารถเริ่มการเข้าถึงหน่วยความจำครั้งถัดไปได้

##Transfer Rate

อัตราความเร็วในการถ่ายโอนข้อมูล

##ลำดับชั้นของหน่วยความจำ (Memory Hierarchy)

เพื่อให้ระบบคอมพิวเตอร์มีประสิทธิภาพสูงและมีต้นทุนเหมาะสม จึงมีการออกแบบหน่วยความจำเป็นลำดับชั้น

###โครงสร้างลำดับชั้นโดยทั่วไปประกอบด้วย

-รีจิสเตอร์

-แคช

-หน่วยความจำหลัก

-หน่วยความจำรอง

-หน่วยความจำสำรองระยะยาว

-คุณสมบัติของลำดับชั้นหน่วยความจำคือ

-หน่วยความจำที่อยู่ใกล้ CPU มากจะมีความเร็วสูง

-ความจุจะเพิ่มขึ้นเมื่ออยู่ลำดับล่าง

ราคาต่อบิตจะลดลง

แนวคิดนี้ช่วยให้ระบบคอมพิวเตอร์สามารถทำงานได้อย่างมีประสิทธิภาพ โดยใช้หน่วยความจำความเร็วสูงเฉพาะส่วนที่จำเป็น

ตอนต่อไปจะเริ่มอธิบายหัวข้อสำคัญของบทนี้ คือ หลักการทำงานของหน่วยความจำแคช

##4.2 หลักการของหน่วยความจำแคช (Cache Memory Principles)

หน่วยความจำแคชถูกออกแบบมาเพื่อแก้ปัญหาความแตกต่างของความเร็วระหว่าง CPU และหน่วยความจำหลัก

แม้ว่า CPU จะสามารถประมวลผลคำสั่งได้อย่างรวดเร็ว แต่การเข้าถึงข้อมูลจากหน่วยความจำหลักมักใช้เวลานานกว่า ดังนั้นจึงมีการเพิ่มหน่วยความจำขนาดเล็กที่มีความเร็วสูงระหว่าง CPU และหน่วยความจำหลัก

หน่วยความจำนี้เรียกว่า Cache Memory

แคชจะเก็บข้อมูลที่ถูกใช้งานบ่อยไว้ใกล้กับ CPU เพื่อให้สามารถเข้าถึงได้รวดเร็วขึ้น

แนวคิด Locality of Reference

การทำงานของแคชอาศัยหลักการสำคัญที่เรียกว่า Locality of Reference

หลักการนี้ระบุว่า โปรแกรมส่วนใหญ่มีแนวโน้มที่จะเข้าถึงข้อมูลในลักษณะที่ซ้ำหรืออยู่ใกล้กัน

Locality แบ่งออกเป็นสองประเภท

Temporal Locality

Spatial Locality

Temporal Locality

หากข้อมูลหนึ่งถูกใช้งานในช่วงเวลาหนึ่ง มีแนวโน้มสูงที่จะถูกใช้งานอีกครั้งในอนาคตอันใกล้

ตัวอย่างเช่น ตัวแปรในลูปของโปรแกรม

Spatial Locality

หากมีการเข้าถึงตำแหน่งหนึ่งของหน่วยความจำ มีแนวโน้มสูงที่จะเข้าถึงตำแหน่งที่อยู่ใกล้เคียง

ตัวอย่างเช่น การอ่านคำสั่งโปรแกรมที่เรียงต่อกันในหน่วยความจำ

4.2 หลักการของหน่วยความจำแคช (Cache Memory Principles)

หน่วยความจำแคชถูกออกแบบขึ้นเพื่อแก้ไขปัญหาความแตกต่างด้านความเร็วระหว่างหน่วยประมวลผลกลาง (CPU) และหน่วยความจำหลัก (Main Memory) ซึ่งเป็นปัญหาที่สำคัญในสถาปัตยกรรมคอมพิวเตอร์สมัยใหม่

โดยทั่วไป CPU สามารถประมวลผลคำสั่งได้รวดเร็วกว่าความสามารถในการส่งข้อมูลของหน่วยความจำหลักอย่างมาก หาก CPU ต้องรอข้อมูลจากหน่วยความจำหลักทุกครั้ง ประสิทธิภาพของระบบจะลดลงอย่างมาก

เพื่อแก้ปัญหานี้จึงมีการเพิ่มหน่วยความจำขนาดเล็กที่มีความเร็วสูงระหว่าง CPU และหน่วยความจำหลัก ซึ่งเรียกว่า หน่วยความจำแคช (Cache Memory)

แนวคิดพื้นฐานของการทำงานของแคช

หลักการสำคัญของหน่วยความจำแคชคือการเก็บสำเนาของข้อมูลที่ถูกใช้งานบ่อยไว้ในหน่วยความจำที่มีความเร็วสูง เพื่อให้ CPU สามารถเข้าถึงข้อมูลเหล่านั้นได้รวดเร็วขึ้น

การทำงานของแคชโดยทั่วไปมีลักษณะดังนี้

CPU ร้องขอข้อมูลจากหน่วยความจำ

ระบบตรวจสอบก่อนว่าข้อมูลอยู่ในแคชหรือไม่

หากพบข้อมูลในแคช CPU จะอ่านข้อมูลจากแคชทันที

หากไม่พบข้อมูล ระบบจะอ่านข้อมูลจากหน่วยความจำหลักแล้วนำมาเก็บไว้ในแคช

กระบวนการดังกล่าวช่วยลดเวลาที่ CPU ต้องรอข้อมูลจากหน่วยความจำหลัก

Cache Hit และ Cache Miss

เมื่อ CPU เข้าถึงข้อมูลในระบบแคช จะเกิดเหตุการณ์ได้สองกรณี

Cache Hit

Cache Miss

Cache Hit

Cache Hit หมายถึงกรณีที่ข้อมูลที่ CPU ต้องการมีอยู่ในหน่วยความจำแคชอยู่แล้ว

ในกรณีนี้ CPU สามารถอ่านข้อมูลจากแคชได้ทันที ทำให้เวลาการเข้าถึงข้อมูลสั้นมาก

Cache Miss

Cache Miss หมายถึงกรณีที่ข้อมูลที่ต้องการไม่อยู่ในแคช

ในกรณีนี้ระบบจะต้อง

เข้าถึงหน่วยความจำหลัก

นำข้อมูลเข้าสู่แคช

ส่งข้อมูลให้ CPU

กระบวนการนี้ใช้เวลานานกว่า Cache Hit อย่างมาก

ขั้นตอนการทำงานของระบบแคช

การทำงานของแคชโดยทั่วไปสามารถอธิบายได้เป็นขั้นตอนดังนี้

CPU ส่งคำสั่งอ่านหรือเขียนข้อมูล

ตัวควบคุมแคชตรวจสอบว่าแอดเดรสของข้อมูลอยู่ในแคชหรือไม่

หากพบข้อมูลในแคช จะเกิด Cache Hit

หากไม่พบ จะเกิด Cache Miss และต้องโหลดข้อมูลจากหน่วยความจำหลัก

เมื่อโหลดข้อมูลแล้ว ระบบจะนำข้อมูลนั้นเก็บไว้ในแคชเพื่อใช้ในอนาคต

แนวคิด Locality of Reference

การออกแบบหน่วยความจำแคชอาศัยแนวคิดสำคัญที่เรียกว่า Locality of Reference

แนวคิดนี้อธิบายพฤติกรรมการเข้าถึงข้อมูลของโปรแกรมคอมพิวเตอร์ ซึ่งมักจะมีรูปแบบการเข้าถึงข้อมูลที่ซ้ำหรืออยู่ใกล้กัน

Locality สามารถแบ่งออกได้เป็นสองประเภทหลัก

Temporal Locality

Spatial Locality

Temporal Locality

Temporal Locality หมายถึงแนวโน้มที่ข้อมูลหรือคำสั่งที่ถูกใช้งานในช่วงเวลาหนึ่ง มีโอกาสสูงที่จะถูกใช้งานอีกครั้งในอนาคตอันใกล้

ตัวอย่างเช่น

ตัวแปรในลูป

ตัวนับในโปรแกรม

คำสั่งที่อยู่ในโครงสร้างวนซ้ำ

การเก็บข้อมูลเหล่านี้ไว้ในแคชช่วยลดเวลาในการเข้าถึงข้อมูล

Spatial Locality

Spatial Locality หมายถึงแนวโน้มที่เมื่อมีการเข้าถึงตำแหน่งหนึ่งของหน่วยความจำ มักจะมีการเข้าถึงตำแหน่งที่อยู่ใกล้เคียงกันในเวลาไม่นาน

ตัวอย่างเช่น

การอ่านคำสั่งของโปรแกรมที่เรียงต่อกัน

Hit Ratio

Hit Ratio หมายถึงอัตราส่วนของจำนวนครั้งที่ข้อมูลถูกพบในแคช

สูตร

Hit Ratio
= จำนวน Cache Hit / จำนวนการเข้าถึงทั้งหมด

ค่า Hit Ratio ที่สูงหมายถึงระบบแคชมีประสิทธิภาพสูง

Miss Ratio

Miss Ratio คืออัตราส่วนของจำนวนครั้งที่ไม่พบข้อมูลในแคช

สูตร

Miss Ratio
= 1 − Hit Ratio

Average Memory Access Time

เวลาการเข้าถึงหน่วยความจำโดยเฉลี่ยสามารถคำนวณได้จาก

Average Access Time
= Hit Time + (Miss Ratio × Miss Penalty)

สูตรนี้ช่วยในการวิเคราะห์ผลกระทบของระบบแคชต่อประสิทธิภาพของ CPU
