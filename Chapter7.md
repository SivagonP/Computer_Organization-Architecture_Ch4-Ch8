# การออกแบบระบบ I/O

- นอกจาก **CPU (Processor)** และ **หน่วยความจำ (Memory Modules)** แล้ว  
  ส่วนสำคัญที่สามของระบบคอมพิวเตอร์คือ **โมดูล I/O**

- โมดูล I/O แต่ละตัวจะ
  - เชื่อมต่อกับ **system bus** หรือ **central switch**
  - ควบคุม **อุปกรณ์ต่อพ่วง (peripheral devices)** อย่างน้อยหนึ่งตัว

- โมดูล I/O ไม่ได้เป็นแค่ตัวเชื่อมสายไฟธรรมดา  
  แต่ภายในมี **วงจรตรรกะ (logic)** ที่ทำหน้าที่สื่อสารระหว่าง
  - อุปกรณ์ภายนอก
  - ระบบบัสของคอมพิวเตอร์

---

# ทำไมไม่เชื่อมต่ออุปกรณ์ภายนอกกับ System Bus โดยตรง

- อุปกรณ์ต่อพ่วงมีหลายชนิด และมีวิธีการทำงานต่างกัน  
- ถ้าให้ CPU ควบคุมทุกอุปกรณ์โดยตรง จะทำให้ระบบซับซ้อนมาก

เหตุผลหลัก ได้แก่

- อุปกรณ์ต่อพ่วงมีวิธีทำงานหลากหลาย  
  - การใส่ logic ทั้งหมดไว้ใน CPU ไม่เหมาะสม

- ความเร็วการส่งข้อมูลของอุปกรณ์ต่อพ่วง  
  - มัก **ช้ากว่า CPU และหน่วยความจำ**

- อุปกรณ์บางชนิด  
  - อาจ **เร็วกว่า CPU หรือหน่วยความจำ**

- อุปกรณ์ต่อพ่วง
  - ใช้ **รูปแบบข้อมูล (data format)** ต่างจากคอมพิวเตอร์
  - ใช้ **ความยาวคำ (word length)** ต่างกัน

# หน้าที่หลักของ I/O Module

โมดูล I/O มีหน้าที่หลัก 2 อย่าง

- เชื่อมต่อกับ **Processor และ Memory**
  - ผ่าน **System Bus หรือ Central Switch**

- เชื่อมต่อกับ **อุปกรณ์ต่อพ่วง**
  - ผ่านช่องสื่อสารข้อมูลที่ออกแบบเฉพาะ

# ลำดับเนื้อหาในบทนี้

- เริ่มจากอธิบาย **อุปกรณ์ภายนอก (External Devices)**

- และวิธีการถ่ายโอนข้อมูลระหว่าง
