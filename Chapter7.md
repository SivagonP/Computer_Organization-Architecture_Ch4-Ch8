# การออกแบบระบบ I/O (I/O System Design)

นอกจาก **หน่วยประมวลผล (Processor)** และ **หน่วยความจำ (Memory Modules)** แล้ว  
องค์ประกอบสำคัญลำดับที่สามของระบบคอมพิวเตอร์คือ

- **ชุดของโมดูล I/O**

โมดูล I/O แต่ละตัวจะ:

- เชื่อมต่อกับ **system bus** หรือ **central switch**
- ควบคุม **อุปกรณ์ต่อพ่วง (peripheral devices)** หนึ่งตัวหรือมากกว่า

โมดูล I/O ไม่ได้เป็นเพียงตัวเชื่อมต่อทางกายภาพ  
ที่ใช้เสียบอุปกรณ์เข้ากับ system bus เท่านั้น

แต่ภายในโมดูล I/O ยังมี

- วงจรตรรกะ (logic circuits)

ที่ทำหน้าที่สื่อสารระหว่าง

- อุปกรณ์ต่อพ่วง
- system bus ของคอมพิวเตอร์

---

# ทำไมไม่เชื่อมต่ออุปกรณ์ต่อพ่วงเข้ากับ System Bus โดยตรง

ผู้อ่านอาจสงสัยว่า  
ทำไมเราไม่เชื่อมต่อ **อุปกรณ์ต่อพ่วง (peripherals)** เข้ากับ **system bus** โดยตรง

เหตุผลมีดังนี้

- อุปกรณ์ต่อพ่วงมีหลายประเภท และมีวิธีการทำงานแตกต่างกันมาก  
  - จึงไม่เหมาะที่จะรวม logic การควบคุมอุปกรณ์ทั้งหมดไว้ใน CPU

- อัตราการถ่ายโอนข้อมูลของอุปกรณ์ต่อพ่วง  
  - มัก **ช้ากว่า CPU และหน่วยความจำ**

- ในทางกลับกัน อุปกรณ์บางชนิด  
  - อาจมีความเร็ว **สูงกว่า CPU หรือหน่วยความจำ**

- อุปกรณ์ต่อพ่วงมักใช้
  - **รูปแบบข้อมูล (data format)** ที่แตกต่างจากคอมพิวเตอร์
  - **ขนาดคำข้อมูล (word length)** ที่แตกต่างกัน

ดังนั้นจึงจำเป็นต้องมี **โมดูล I/O**

---

# หน้าที่หลักของ I/O Module

โมดูล I/O มีหน้าที่สำคัญ 2 ประการ

- เชื่อมต่อกับ **Processor และ Memory**
  - ผ่าน **system bus หรือ central switch**

- เชื่อมต่อกับ **อุปกรณ์ต่อพ่วง**
  - ผ่านช่องทางการสื่อสารข้อมูลที่ออกแบบเฉพาะสำหรับอุปกรณ์นั้น

---

# แบบจำลองทั่วไปของ I/O Module

องค์ประกอบหลักของโมดูล I/O ได้แก่

- Address lines
- Data lines
- Control lines
- System bus
- I/O module
- การเชื่อมต่อไปยังอุปกรณ์ต่อพ่วง

โมดูล I/O ทำหน้าที่เป็น **ตัวกลางระหว่างระบบคอมพิวเตอร์และอุปกรณ์ภายนอก**

# โครงสร้างเนื้อหาในบทนี้

บทนี้จะเริ่มจาก

- การอธิบาย **อุปกรณ์ภายนอก (External Devices)**

จากนั้นจะอธิบาย

- โครงสร้างและหน้าที่ของ **I/O Module**

ต่อมาจะกล่าวถึงวิธีต่าง ๆ ในการถ่ายโอนข้อมูลระหว่าง

- CPU
- หน่วยความจำ
- อุปกรณ์ I/O

หัวข้อสำคัญที่กล่าวถึง ได้แก่

- Programmed I/O
- Interrupt-Driven I/O
- Direct Memory Access (DMA)
- Direct Cache Access

และในตอนท้ายจะกล่าวถึง

- การเชื่อมต่อ I/O ภายนอก
- การเชื่อมต่อระหว่าง **โมดูล I/O กับอุปกรณ์ภายนอก**
  # 7.1 อุปกรณ์ภายนอก (External Devices)

อุปกรณ์ภายนอก หรือ **Peripheral Devices** เป็นอุปกรณ์ที่เชื่อมต่อกับระบบคอมพิวเตอร์  
เพื่อใช้สำหรับ **รับข้อมูล (Input)**, **ส่งออกข้อมูล (Output)** หรือ **จัดเก็บข้อมูล (Storage)**

อุปกรณ์เหล่านี้มักถูกควบคุมผ่าน **I/O Module**

---

## ประเภทของอุปกรณ์ภายนอก

โดยทั่วไปสามารถแบ่งอุปกรณ์ภายนอกออกเป็น 3 ประเภทหลัก

- **Human Readable Devices**
- **Machine Readable Devices**
- **Communication Devices**

---

## Human Readable Devices

อุปกรณ์ที่มนุษย์สามารถใช้งานหรือเข้าใจได้โดยตรง

ตัวอย่างเช่น

- Keyboard
- Mouse
- Monitor
- Printer

อุปกรณ์ประเภทนี้ทำหน้าที่เป็นตัวกลางระหว่าง

- ผู้ใช้
- ระบบคอมพิวเตอร์

---

## Machine Readable Devices

อุปกรณ์ที่ใช้สำหรับการสื่อสารกับเครื่องจักรหรือระบบอื่น

ตัวอย่างเช่น

- Magnetic disk
- USB drive
- Sensors
- Controllers

อุปกรณ์เหล่านี้มักใช้ใน

- ระบบควบคุมอัตโนมัติ
- ระบบอุตสาหกรรม
- ระบบฝังตัว (Embedded Systems)

---

## Communication Devices

อุปกรณ์ที่ใช้สำหรับการสื่อสารข้อมูลระหว่างคอมพิวเตอร์

ตัวอย่างเช่น

- Network Interface Card (NIC)
- Modem
- Wireless adapters

อุปกรณ์เหล่านี้ช่วยให้คอมพิวเตอร์สามารถ

- เชื่อมต่อเครือข่าย
- ส่งข้อมูลระหว่างเครื่อง
- ติดต่อกับระบบอื่น

---

# ความแตกต่างของอุปกรณ์ภายนอก

อุปกรณ์ภายนอกมีความแตกต่างกันหลายด้าน เช่น

- อัตราความเร็วในการรับส่งข้อมูล
- รูปแบบข้อมูล
- วิธีควบคุมการทำงาน
- รูปแบบการสื่อสาร

เนื่องจากความแตกต่างเหล่านี้  
จึงต้องใช้ **I/O Module** เพื่อจัดการความแตกต่างระหว่าง

- ระบบคอมพิวเตอร์
- อุปกรณ์ต่อพ่วง

---

# 7.2 I/O Modules

**I/O Module** เป็นองค์ประกอบสำคัญที่ทำหน้าที่

- เชื่อมต่ออุปกรณ์ภายนอกกับระบบคอมพิวเตอร์
- ควบคุมการถ่ายโอนข้อมูล
- แปลงรูปแบบข้อมูลระหว่างอุปกรณ์และระบบ

---

## หน้าที่ของ I/O Module

หน้าที่หลักของ I/O Module ได้แก่

- Control and Timing
- Processor Communication
- Device Communication
- Data Buffering
- Error Detection

---

## Control and Timing

I/O Module ทำหน้าที่ควบคุม

- ลำดับการทำงานของอุปกรณ์
- เวลาในการรับส่งข้อมูล

เพื่อให้การทำงานระหว่าง

- CPU
- Memory
- Peripheral Devices

เป็นไปอย่างถูกต้อง

---

## Processor Communication

I/O Module ต้องสามารถสื่อสารกับ CPU ได้

การสื่อสารนี้ประกอบด้วย

- การรับคำสั่งจาก CPU
- การส่งสถานะกลับไปยัง CPU
- การรับหรือส่งข้อมูล

ตัวอย่างคำสั่งที่ CPU อาจส่งให้ I/O Module เช่น

- Read
- Write
- Control
- Test

---

## Device Communication

I/O Module ยังต้องสามารถสื่อสารกับ

- อุปกรณ์ต่อพ่วง

โดยต้องสามารถ

- ส่งคำสั่งไปยังอุปกรณ์
- รับข้อมูลจากอุปกรณ์
- รับสถานะการทำงานของอุปกรณ์

---

## Data Buffering

เนื่องจาก

- ความเร็วของ CPU สูง
- ความเร็วของอุปกรณ์ต่อพ่วงต่ำกว่า

I/O Module จึงต้องมี **Buffer**

เพื่อเก็บข้อมูลชั่วคราวระหว่างการถ่ายโอน

การใช้ Buffer ช่วยให้

- ลดความแตกต่างของความเร็ว
- เพิ่มประสิทธิภาพของระบบ

---

## Error Detection

I/O Module ยังมีหน้าที่

- ตรวจสอบข้อผิดพลาดในการรับส่งข้อมูล

ตัวอย่างเช่น

- parity error
- transmission error

เมื่อพบข้อผิดพลาด

ระบบจะสามารถ

- แจ้งเตือน CPU
- ทำการแก้ไขหรือส่งข้อมูลใหม่
  # 7.3 Programmed I/O

Programmed I/O เป็นวิธีการถ่ายโอนข้อมูลระหว่าง  
**CPU และอุปกรณ์ I/O** โดยที่ **CPU เป็นผู้ควบคุมการทำงานทั้งหมด**

ในวิธีนี้ CPU จะต้อง

- ส่งคำสั่งไปยัง I/O module
- ตรวจสอบสถานะของอุปกรณ์
- รับหรือส่งข้อมูลด้วยตัวเอง

---

## ขั้นตอนการทำงานของ Programmed I/O

ขั้นตอนทั่วไปมีดังนี้

- CPU ส่งคำสั่ง I/O ไปยัง I/O module
- I/O module ตรวจสอบสถานะของอุปกรณ์
- CPU ตรวจสอบสถานะของ I/O module อย่างต่อเนื่อง
- เมื่ออุปกรณ์พร้อม CPU จะทำการรับหรือส่งข้อมูล

วิธีนี้เรียกว่า **Polling**

---

## Polling

Polling คือการที่ CPU

- ตรวจสอบสถานะของอุปกรณ์ I/O ซ้ำ ๆ อย่างต่อเนื่อง

CPU จะถามอุปกรณ์ว่า

- พร้อมทำงานหรือยัง

ถ้าอุปกรณ์ยังไม่พร้อม  
CPU จะต้องตรวจสอบซ้ำไปเรื่อย ๆ

---

## ข้อเสียของ Programmed I/O

Programmed I/O มีข้อเสียสำคัญคือ

- CPU ต้องเสียเวลาในการตรวจสอบอุปกรณ์ตลอดเวลา
- ทำให้ CPU ไม่สามารถไปทำงานอื่นได้
- ประสิทธิภาพของระบบลดลง

ดังนั้นจึงมีการพัฒนาเทคนิคอื่น เช่น

- Interrupt-Driven I/O

---

# 7.4 Interrupt-Driven I/O

Interrupt-Driven I/O เป็นวิธีที่ช่วยลดภาระของ CPU

แทนที่ CPU จะต้องตรวจสอบอุปกรณ์ตลอดเวลา  
อุปกรณ์จะ **ส่งสัญญาณ Interrupt ไปยัง CPU เมื่อพร้อมทำงาน**

---

## หลักการทำงานของ Interrupt

ขั้นตอนการทำงานโดยทั่วไป

- CPU ส่งคำสั่ง I/O ไปยัง I/O module
- CPU ไปทำงานอื่นต่อ
- เมื่ออุปกรณ์พร้อม จะส่ง **Interrupt Signal**
- CPU หยุดการทำงานชั่วคราว
- CPU ทำการรับหรือส่งข้อมูลกับอุปกรณ์
- จากนั้น CPU กลับไปทำงานเดิม

---

## Interrupt Handler

เมื่อเกิด Interrupt

CPU จะเรียกใช้โปรแกรมพิเศษที่เรียกว่า

- **Interrupt Handler**
- หรือ **Interrupt Service Routine (ISR)**

หน้าที่ของโปรแกรมนี้คือ

- จัดการกับเหตุการณ์ interrupt
- รับหรือส่งข้อมูลจากอุปกรณ์
- คืนการควบคุมกลับไปยังโปรแกรมหลัก

---

## ข้อดีของ Interrupt-Driven I/O

ข้อดีของวิธีนี้คือ

- CPU ไม่ต้องเสียเวลาตรวจสอบอุปกรณ์ตลอดเวลา
- CPU สามารถทำงานอื่นได้
- ประสิทธิภาพของระบบดีขึ้น

อย่างไรก็ตาม  
CPU ยังคงต้องมีส่วนร่วมในการถ่ายโอนข้อมูล

---

# 7.5 Direct Memory Access (DMA)

**Direct Memory Access (DMA)** เป็นเทคนิคที่ช่วยให้

- อุปกรณ์ I/O สามารถถ่ายโอนข้อมูลไปยังหน่วยความจำได้โดยตรง

โดยไม่ต้องให้ CPU ควบคุมทุกขั้นตอน

---

## หลักการของ DMA

ในการทำงานแบบ DMA

- CPU จะตั้งค่าการถ่ายโอนข้อมูล
- จากนั้น **DMA Controller** จะควบคุมการถ่ายโอนข้อมูลเอง

ขั้นตอนโดยทั่วไป

- CPU ส่งคำสั่งให้ DMA controller
- DMA controller ติดต่อกับอุปกรณ์ I/O
- DMA controller ถ่ายโอนข้อมูลระหว่าง  
  - อุปกรณ์ I/O  
  - หน่วยความจำ

โดยไม่ต้องให้ CPU จัดการข้อมูลทีละคำ

---

## ข้อดีของ DMA

ข้อดีของ DMA ได้แก่

- ลดภาระการทำงานของ CPU
- เพิ่มความเร็วในการถ่ายโอนข้อมูล
- ทำให้ CPU สามารถทำงานอื่นได้

---

## DMA Controller

DMA Controller เป็นฮาร์ดแวร์ที่ทำหน้าที่

- ควบคุมการถ่ายโอนข้อมูลระหว่าง
  - I/O device
  - Memory

DMA controller จะมีข้อมูลสำคัญ เช่น

- Memory address
- จำนวนข้อมูลที่จะถ่ายโอน
- ทิศทางของการถ่ายโอนข้อมูล

เมื่อการถ่ายโอนเสร็จสิ้น  
DMA controller จะส่ง **Interrupt** ไปยัง CPU
เพื่อแจ้งว่าการทำงานเสร็จแล้ว
# โหมดการทำงานของ DMA (DMA Modes)

การทำงานของ **Direct Memory Access (DMA)** สามารถแบ่งออกเป็นหลายโหมด  
ซึ่งกำหนดวิธีที่ DMA ใช้ **system bus** ร่วมกับ CPU

โหมดหลัก ๆ ได้แก่

- Burst Mode
- Cycle Stealing
- Transparent Mode

---

## Burst Mode

Burst Mode เป็นโหมดที่

- DMA จะควบคุม system bus ทั้งหมด
- ทำการถ่ายโอนข้อมูลต่อเนื่องหลายคำ

ในช่วงเวลานี้

- CPU จะไม่สามารถใช้งาน system bus ได้

ข้อดี

- การถ่ายโอนข้อมูลรวดเร็วมาก

ข้อเสีย

- CPU ต้องหยุดการเข้าถึงหน่วยความจำชั่วคราว

---

## Cycle Stealing

Cycle Stealing เป็นโหมดที่ DMA

- แย่งการใช้งาน system bus จาก CPU  
  ทีละรอบ (cycle)

DMA จะ

- ถ่ายโอนข้อมูลทีละคำ
- จากนั้นคืน bus ให้ CPU

ข้อดี

- CPU ยังคงสามารถทำงานได้

ข้อเสีย

- ความเร็วในการถ่ายโอนข้อมูลต่ำกว่า Burst Mode

---

## Transparent Mode

Transparent Mode เป็นโหมดที่ DMA

- ใช้ system bus เฉพาะตอนที่ CPU ไม่ได้ใช้งาน

ข้อดี

- CPU จะไม่ได้รับผลกระทบจาก DMA

ข้อเสีย

- การถ่ายโอนข้อมูลอาจช้ากว่าโหมดอื่น

---

# 7.6 Direct Cache Access

**Direct Cache Access (DCA)** เป็นเทคนิคที่ช่วยให้

- อุปกรณ์ I/O สามารถส่งข้อมูลไปยัง **Cache Memory** ได้โดยตรง

แทนที่จะส่งไปยัง **Main Memory** ก่อน

---

## ข้อดีของ Direct Cache Access

ข้อดีของ DCA ได้แก่

- ลดเวลาในการเข้าถึงข้อมูล
- ลดภาระของหน่วยความจำหลัก
- เพิ่มประสิทธิภาพของระบบโดยรวม

เทคนิคนี้มีประโยชน์มากในระบบที่ต้องการ

- การประมวลผลข้อมูลความเร็วสูง
- การรับส่งข้อมูลเครือข่ายจำนวนมาก

---

# 7.7 I/O Channels และ I/O Processors

ในระบบคอมพิวเตอร์ขนาดใหญ่  
การจัดการ I/O อาจซับซ้อนมาก

จึงมีการใช้

- **I/O Channels**
- **I/O Processors**

เพื่อช่วยจัดการงานด้าน I/O

---

## I/O Channels

I/O Channel เป็นหน่วยฮาร์ดแวร์ที่

- จัดการการถ่ายโอนข้อมูลระหว่าง
  - Memory
  - I/O Devices

โดยลดภาระของ CPU

CPU จะเพียง

- ส่งคำสั่งระดับสูง
- แล้ว I/O Channel จะจัดการรายละเอียดเอง

---

## ประเภทของ I/O Channel

I/O Channel สามารถแบ่งออกเป็นหลายประเภท เช่น

- Selector Channel
- Multiplexer Channel

---

### Selector Channel

Selector Channel เป็นช่องสัญญาณที่

- จัดการกับอุปกรณ์ I/O ทีละตัว

เหมาะกับอุปกรณ์ที่ต้องการ

- ความเร็วสูง
- การถ่ายโอนข้อมูลจำนวนมาก

ตัวอย่างเช่น

- Disk drives

---

### Multiplexer Channel

Multiplexer Channel สามารถ

- จัดการอุปกรณ์หลายตัวพร้อมกัน

โดยแบ่งเวลาในการใช้งาน

เหมาะกับอุปกรณ์ที่

- มีความเร็วต่ำ
- ส่งข้อมูลเป็นช่วง ๆ

ตัวอย่างเช่น

- Keyboard
- Printer
- Terminals

---

## I/O Processor

**I/O Processor (IOP)** เป็นโปรเซสเซอร์พิเศษที่ทำหน้าที่

- ควบคุมและจัดการงานด้าน I/O

IOP สามารถ

- ดำเนินคำสั่ง I/O ด้วยตัวเอง
- จัดการอุปกรณ์หลายตัวพร้อมกัน

ข้อดี

- ลดภาระการทำงานของ CPU
- เพิ่มประสิทธิภาพของระบบ

CPU จะทำหน้าที่เพียง

- เริ่มต้นงาน I/O
- และรับผลลัพธ์เมื่อการทำงานเสร็จ
  # 7.8 External I/O Interfaces

การเชื่อมต่อระหว่าง **โมดูล I/O** กับ **อุปกรณ์ภายนอก**  
จำเป็นต้องมีมาตรฐานการสื่อสารที่กำหนดวิธีการส่งข้อมูล

มาตรฐานเหล่านี้เรียกว่า

- **External I/O Interfaces**

อินเทอร์เฟซเหล่านี้กำหนด

- วิธีการเชื่อมต่อทางกายภาพ
- รูปแบบของสัญญาณไฟฟ้า
- โปรโตคอลการสื่อสารข้อมูล

---

## องค์ประกอบของ I/O Interface

อินเทอร์เฟซ I/O โดยทั่วไปประกอบด้วย

- Mechanical specification  
- Electrical specification  
- Functional specification  
- Procedural specification

---

### Mechanical Specification

กำหนดลักษณะทางกายภาพของการเชื่อมต่อ เช่น

- รูปร่างของหัวต่อ (connector)
- จำนวนขา (pins)
- รูปแบบสายสัญญาณ

---

### Electrical Specification

กำหนดลักษณะของสัญญาณไฟฟ้า เช่น

- ระดับแรงดันไฟฟ้า
- การเข้ารหัสสัญญาณ
- วิธีการส่งสัญญาณ

---

### Functional Specification

กำหนดหน้าที่ของสัญญาณต่าง ๆ เช่น

- สัญญาณข้อมูล
- สัญญาณควบคุม
- สัญญาณสถานะ

---

### Procedural Specification

กำหนดลำดับขั้นตอนของการสื่อสารข้อมูล เช่น

- วิธีการเริ่มต้นการส่งข้อมูล
- การตอบรับข้อมูล
- การสิ้นสุดการส่งข้อมูล

---

# ตัวอย่างมาตรฐาน I/O Interface

ตัวอย่างของมาตรฐานการเชื่อมต่อ I/O ที่ใช้กันทั่วไป เช่น

- USB (Universal Serial Bus)
- SATA (Serial ATA)
- PCI Express
- Thunderbolt
- Ethernet

มาตรฐานเหล่านี้ช่วยให้

- อุปกรณ์จากผู้ผลิตต่าง ๆ สามารถทำงานร่วมกันได้

# 7.9 I/O Structure ของ IBM zEnterprise EC12

**IBM zEnterprise EC12** เป็นระบบเมนเฟรมที่ออกแบบมาเพื่อ

- รองรับงานประมวลผลขนาดใหญ่
- การจัดการข้อมูลจำนวนมาก

ระบบนี้มีโครงสร้าง I/O ที่มีประสิทธิภาพสูง

---

## ลักษณะสำคัญของ I/O ในระบบนี้

ระบบ I/O ของ zEnterprise EC12 มีลักษณะสำคัญ เช่น

- ใช้ **I/O channels** เพื่อจัดการอุปกรณ์จำนวนมาก
- ใช้ **channel subsystem**
- รองรับ **การทำงานแบบขนาน**

สิ่งเหล่านี้ช่วยให้ระบบสามารถ

- จัดการอุปกรณ์ I/O จำนวนมากได้อย่างมีประสิทธิภาพ

---

## Channel Subsystem

Channel subsystem เป็นส่วนที่ทำหน้าที่

- ควบคุมการสื่อสารระหว่าง
  - CPU
  - I/O devices

หน้าที่ของ subsystem นี้ ได้แก่

- จัดการคำสั่ง I/O
- ควบคุมการถ่ายโอนข้อมูล
- จัดการสถานะของอุปกรณ์
